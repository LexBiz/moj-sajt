## Onboarding клиента (как это будет у Вас работать) — пошагово

Ниже — “конвейер”, который Вы повторяете для каждого клиента. Цель: **подключили → протестировали → запустили** за минимальное число шагов.

### Шаг 1. Создать клиента (tenant)
В админке:
- Название/бренд
- Пакет (START / BUSINESS / PRO)
- Язык по умолчанию (обычно UA)
- Таймзона
- Куда слать уведомления (Telegram менеджера/владельца)

Результат: появляется `tenantId`.

### Шаг 2. Выбрать и подключить каналы (wizard)
Для каждого канала — кнопка “Connect”:
- Instagram Direct
- Instagram Comments
- WhatsApp
- Telegram bot
- Website chat / Flow

Система должна:
- провести OAuth (если применимо)
- сохранить токены/ID в `channel_connections`
- подписать вебхуки/подписки (где возможно)
- показать “зелёный статус” + тестовую кнопку

### Шаг 3. Заполнить “Профиль бизнеса” (самое важное)
Это “мозг” под нишу:
- Ниша + география
- Услуги + предложения
- FAQ / возражения
- Правила лид-сбора (обязательно телефон+email, когда спрашивать)
- Тон по каналам (Instagram живее, WhatsApp спокойнее и т.д.)
- Языковая политика (UA default, RU только по явному запросу)

### Шаг 4. Включить сценарии (toggle)
Примеры сценариев:
- IG comments: “+” → comment reply + DM first message template
- IG comments: “цена” → “написал в Direct ✅” + DM
- Follow-up через 3 часа (только если лид не собран и в окне 24h)
- “Кто вы/что вы?” без просьбы контакта

### Шаг 5. Тест
В админке должны быть тесты:
- “Send test DM”
- “Simulate comment event”
- “Check last comment replies”
- “Reset convo”
- “Health”

Цель теста:
- сообщение пришло
- бот ответил
- лид создался/обновился
- уведомление ушло

### Шаг 6. Go Live
Включаем “боевой режим”:
- вебхуки включены
- сценарии включены
- мониторинг включен

### Шаг 7. После запуска (операционка)
По пакету:
- START: контроль работоспособности + мелкие правки текста
- BUSINESS: оптимизация вопросов/логики, аналитика
- PRO: интеграции/приоритет/ежемесячные улучшения
