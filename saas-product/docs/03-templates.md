## Шаблоны (prompts) и как их не превратить в кашу

Цель: “идеальный ассистент” легко переносится на других клиентов **без переписывания ядра**.

### Правило №1: промпт — слоёный
Не один монолитный текст, а 4 слоя:

1) **Core (общий для всех)**  
Роль/качество/безопасность/воронка.

2) **Channel Tone (на 5 каналов)**  
Instagram = живее/короче/эмодзи, WhatsApp = спокойнее, Telegram = быстро/делово, Website/Flow = структурнее.

3) **Client Profile (для конкретного клиента)**  
Ниша, услуги, FAQ, условия, гео, офферы, прайс.

4) **Scenario / Trigger (по событию)**  
Например:
- IG comment “+” → first DM template (не продаём, не просим контакт)
- IG comment “цена” → “написал в Direct ✅” + DM
- follow-up 3h → одно короткое сообщение
- “кто вы?” → отвечаем без просьбы контакта

### Правило №2: цены/пакеты — НЕ в промпте строкой
Прайс (START/BUSINESS/PRO и доп. модули) хранится как **данные профиля**.
Промпт лишь говорит “используй прайс из профиля”.
Тогда обновление прайса = **одно место**, и работает во всех каналах.

### Правило №3: версии
Для масштабирования обязательно:
- version промпта (например v1/v2)
- журнал изменений “что поменяли и почему”
- быстрый откат
